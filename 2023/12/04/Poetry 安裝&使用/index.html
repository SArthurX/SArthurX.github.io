Poetry 安裝&使用

Created2023-12-04|Updated2024-02-13
|Word Count:1.9k|Reading Time:6mins|Post Views:

最近被pipenv搞到，我原本在用python 3.12，但因為某個必要的套件沒更新上來似乎只能在舊版安裝，想了一想python也裝了3個版本，似乎也沒必要用到那麼多就把最新的3.12和3.11砍了再去安裝穩定版的3.10，原本以為只有單純的改一下環境變數而已沒什麼太大問題，可這時候問題就來了==，後續pipenv不管怎樣install都會吃到舊版本，重下載重開機也沒用，環境也都確認移除掉了，不知道哪裡跑出來原本的3.11

[{"url":"pipenv_error.png","alt":""}]

解決方法

最後發現似乎是pipenv裡的緩存搞的，雖然我也不知道為什麼uninstall掉還會有什麼緩存，總之只要以下指令清除就可以正常安裝了

[{"url":"pipenv_fix.png","alt":""}]

但這也讓我萌生了放棄pipenv的想法，上網查了一下有沒有相關的套件管理工具時發現了 Poetry
這個好用的東西，因為我本來就看pipenv lock時間很不順眼了，以前在玩arch的時候不知道為什麼每次都要lock個十幾年才能轉好== ，即使用--skip-lock時間也不會差多少，總之既然有更好的工具為何不用呢

安裝Poetry

可以用pip安裝或是官方的方式下載 官方安裝Doc
那就使用官方的吧

Linux, macOS, Windows (WSL)

Windows (Powershell)

安裝完後應該會在以下路徑

  • $HOME/.local/bin on Unix
  • %APPDATA%\Python\Scripts on Windows

設定 PATH

Linux

如果選擇「全域安裝」一定要先設置PATH 環境變數才能使用，如果是Linux就只要在shell的config加上

( shell config名稱bash –> .bashrc zsh –> .zshrc 通常會在家目錄底下 )

Windows

我通常都直接搜尋啦就不打指令了

  1. Windows 搜尋「編輯系統環境變數」點進去
  2. 「進階」
  3. 點選「環境變數」
  4. 此時會打開另一個新視窗，有「使用者變數」、「系統變數」 我通常都只改使用者變數而已，就差在其他使用者可不可以用，而系統變數需要重開機才會生效。
  5. 裡面會有一個Path的欄位 點進去
  6. 按新增後把路徑打上去或點瀏覽選擇該路徑，然後點確定就完成了

新增完後別忘了重開你的Terminal

重開後就可以確認看看有沒有成功安裝了

初始化 Poetry

要先初始化才能使用Poetry，就隨便開一個資料夾測試吧

初始化後他會跳出一些選項讓你選，沒什麼差別大部分就先全都enter就好

最後會問你「Do you confirm generation? (yes/no) [yes]」，按enter後就建立完成了，這時候你會看到目錄裡生成了一個pyproject.toml的檔案，這樣就初始化成功了

建立虛擬環境(venv)

為什麼要建立虛擬環境呢，因為不是每個專案都支援相同版本的套件，而直接pip install下來可能會使其他套件產生衝突，這時候我們就需要虛擬環境去幫我們做隔絕的動作，我用下來還是覺得Poetry和Pipenv的環境開啟方式我最喜歡就簡簡單單打一行就可以了，不像其他虛擬環境還需要source資料夾裡面的activate，回歸正題，Poetry和Pipenv的虛擬環境開啟方式一樣

就會幫你開起來了，而通常進入虛擬環境後你的shell也會顯示 (該環境名稱) $ 告訴你目前的狀態是在虛擬環境中，在這環境下執行的操作就不怕去汙染到全域了

如果要建立其他版本的虛擬環境則需要使用

只打python的話他會以你預設的python版本去建立，如果指定其他版本就把後面的python加上該版本就行了

虛擬環境路徑

預設情況建立一個虛擬環境後預設路徑會放在/User/AppData/Local/pypoetry/Cache/virtualenvs 底下，名稱是 專案名稱-亂數-Python版本

雖然沒什麼差但如果想讓虛擬環境跟著專案包在一起的話可以修改Poetry的config
先看看Poetry有哪些config

其中有一項virtualenvs.in-project = null，我們修改為true

就可以把新創建的環境放在專案下了，再來重新建立一個環境就會看到目錄底下生成名稱為.venv的資料夾，這就是我們的環境所在地

安裝及刪除套件

安裝套件

跟pip install一樣只是poetry改成add了

如果在後面加上參數--group dev則會安裝在開發環境上
安裝後pyproject.toml裡面就會加上你安裝的套件名稱及版本

除了pyproject.toml外，專案裡還會多一個poetry.lock的檔案，虛擬環境就是依照這個檔案去做更新的
當你poetry add後，Poetry會自動幫你做完這三件事

  1. 寫入pyproject.toml
  2. 依照pyproject.toml的內容，更新poetry.lock
  3. 再依照poetry.lock，更新虛擬環境

所以如果我們手動更改了pyproject.toml記得使用

讓poetry.lock同步，畢竟虛擬環境是基於他去更新套件的
說白了就是每次Poetry安裝套件時都會幫你自動生成requirements.txt

而要一次安裝pyproject.toml裡的全部套件時只要

就會幫你一次把所有套件安裝下來

移除套件

因為Poetry有解析依賴套件的功能，所以可以放心移除某個專案不要用到的套件

這正是和pip不同的地方，pip的uninstall只會移除掉該套件而已，而不移除該套件的依賴，雖然是基於安全性但也會讓電腦裡裝很多不必要的東西，但如果真的要完全移除依賴套件又可能會造成其他也依賴這套件的套件(怎麼有點繞口)無法使用，改用Poetry後就可以避免在這個問題，移除之前他會檢查其餘套件是否也會使用到這個依賴，如果都沒用到才會刪除，這樣就不會發生可能會用到這個依賴包卻被不小心移除掉了情況了。
